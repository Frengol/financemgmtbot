# Projeto: Finance Mgmt Bot

1. Sempre siga o `architecture.md` deste repositório. Se ele não existir, pare e peça orientação.
2. Para novos desenvolvimentos, escreva primeiro os testes e depois o código.
3. Após implementar, rode imediatamente os testes novos e os regressivos até tudo passar.
4. Faça apenas alterações aprovadas via chat e pare se houver dúvida de escopo.
5. Depois de concluir o código, atualize o `architecture.md` do repositório.
6. Nunca faça commit sem permissão explícita do usuário.
